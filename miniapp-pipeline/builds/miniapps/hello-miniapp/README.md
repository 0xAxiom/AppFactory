# Hello Mini App

A simple example mini app demonstrating the MiniApp Pipeline output structure.

## What is this?

This is a complete example of a Base Mini App generated by the `miniapp-pipeline`. It includes:

- A Next.js application with all required components
- Manifest configuration (`minikit.config.ts`)
- Manifest route (`/.well-known/farcaster.json`)
- Error handling and browser fallback
- Pipeline artifacts showing the generation process

## Structure

```
hello-miniapp/
├── app/                    # The Next.js application
│   ├── app/
│   │   ├── page.tsx       # Main greeting page
│   │   ├── layout.tsx     # Root layout
│   │   └── .well-known/   # Manifest route
│   ├── components/        # UI components
│   ├── public/            # Assets (placeholders)
│   └── minikit.config.ts  # Manifest configuration
└── artifacts/             # Pipeline outputs
    ├── inputs/            # Normalized prompt
    ├── stage01/           # Scaffold plan
    ├── stage02/           # Scaffold complete
    ├── stage03/           # Manifest config
    └── polish/            # Ralph review artifacts
```

## Running Locally

```bash
cd app
npm install
npm run dev
```

Then visit:
- http://localhost:3000 - The app
- http://localhost:3000/.well-known/farcaster.json - The manifest

## Before Publishing

1. **Replace placeholder images** in `app/public/`
2. **Deploy to Vercel**
3. **Complete account association** (Stage M5)
4. **Update minikit.config.ts** with real signature values
5. **Publish** by posting URL in Base app

## Note

This example includes placeholder account association values. You must complete the signing process (Stage M5) before your app will be recognized by Base.

See `artifacts/polish/ralph_final_verdict.md` for the quality review.
