Expo Install --check Behavior

Key Characteristics

  • Helps install React Native packages with version compatibility
  • Ensures packages work with the currently installed version of `react-native`

Version Validation Features

  • --check flag: Validates installed package versions
  • --fix flag: Automatically updates packages to compatible versions

Command Examples

npx expo install expo-camera
npx expo install typescript expo-sms
npx expo install --check
npx expo install --fix

Compatibility Rules

  • Checks package versions against a list of known working combinations
  • Supports multiple package managers: bun, npm, pnpm, yarn
  • Can exclude specific packages from version checking via `expo.install.exclude` in package.json

Package Manager Selection

  • Automatically detects package manager based on lockfile
  • Can force a specific package manager with flags like `--npm`, `--yarn`, etc.

Critical Principle

React Native is not backwards compatible - expo install carefully manages package versions to ensure compatibility.

Expected Version Authority

  • The output of `npx expo install --check` is authoritative
  • Expected versions listed must be respected
  • Mismatches indicate potential runtime issues
  • Use `--fix` to align to expected versions
