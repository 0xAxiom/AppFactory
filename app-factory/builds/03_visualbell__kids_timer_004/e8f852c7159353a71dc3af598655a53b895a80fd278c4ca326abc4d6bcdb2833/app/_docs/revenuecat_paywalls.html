RevenueCat Paywalls and Offerings

Key Concepts

  • Offerings are configurable product groups that can be dynamically updated without app updates
  • Packages abstract platform-specific products across iOS, Android, and web
  • Packages have an identifier, type (e.g., MONTHLY, ANNUAL), and underlying store product

Displaying Products in React Native

try {
  const offerings = await Purchases.getOfferings();
  if (offerings.current && offerings.current.monthly) {
    const product = offerings.current.monthly;
    // Display product details
  }
} catch (error) {
  // Handle errors
}

Best Practices

  • Make paywalls dynamic
  • Use default package types
  • Support variable product choices
  • Allow different free trial configurations

Customization Options

  • Fetch offerings by placement
  • Choose offerings based on user attributes
  • Access custom offering identifiers

Implementation Rules

  • Fetch Offerings at runtime: Always get current offerings from RevenueCat
  • Handle offline states: Provide fallback UI when offerings unavailable
  • Error handling: Gracefully handle network and API errors
  • Dynamic updates: Leverage remote configuration capabilities

Goal

Create flexible, remotely configurable subscription experiences that can adapt to different user segments and marketing strategies.
