RevenueCat Entitlements and Feature Gating

Entitlements Overview

  • Represent a level of access or features users can unlock in an app
  • Scoped to a project and typically unlocked after purchasing a product
  • Most apps have one entitlement, but multiple are possible

Key Characteristics

  • Purpose: "Entitlements are used to ensure a user has appropriate access to content based on their purchases"
  • Shared across all apps in the same project
  • Can be attached to multiple products
  • Products can unlock multiple entitlements

Creating Entitlements

  1. Go to Product catalog > Entitlements tab
  2. Click "+ New entitlement"
  3. Create a unique identifier (e.g., "pro")

Checking Entitlement Status

  • Use RevenueCat SDK's `getCustomerInfo` method
  • If entitlement is active, unlock content
  • If inactive, display a paywall

Feature Gating Implementation Rules

  • Gate by Entitlements only: Never check products directly
  • Check at runtime: Always verify current entitlement status
  • Handle offline: Cache entitlement status for offline access
  • Prevent bypass: No deep-link or navigation bypass of paywalls
  • Graceful fallback: Provide basic functionality if entitlement check fails

Restore Purchases

  • Provide restore purchases UI: Required for App Store compliance
  • Handle restore failures: Network errors, no previous purchases
  • Update entitlements: Refresh customer info after restore

Best Practices

  • Attach relevant products to entitlements
  • Consider product type when mapping entitlements
  • Ensure new products are added to appropriate entitlements
