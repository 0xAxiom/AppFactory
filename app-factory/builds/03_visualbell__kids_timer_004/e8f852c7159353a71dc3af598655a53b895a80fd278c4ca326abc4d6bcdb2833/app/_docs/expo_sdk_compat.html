Expo SDK Compatibility Rules

SDK Upgrade Principles

  • Upgrade SDK versions incrementally, one at a time
  • This helps identify specific breakages during the upgrade process

Upgrade Steps

  1. Install new Expo SDK version:
    • Use package manager command like `npm install expo@^54.0.0`
    • Replace version number with target SDK version
  2. Dependency Alignment:
    • Run `npx expo install --fix` to upgrade dependencies
    • Use `npx expo-doctor` to check for common problems
  3. Native Project Updates:
    • For Continuous Native Generation: Delete android/ios directories
    • Without Continuous Native Generation: Run `npx pod-install`

Key Recommendations

  • Check SDK release notes for specific breaking changes
  • Use development builds for production apps
  • Expo Go only supports the latest SDK version

Compatibility Guidance

  • Incrementally upgrade to pinpoint potential issues
  • Follow release notes for version-specific instructions
  • Verify dependencies match the new SDK version

Hard Gate Rules

  • npx expo install --check output is authoritative
  • Expected versions must be respected
  • Do NOT silence errors with arbitrary resolutions
  • Do NOT upgrade Expo to chase dependencies
  • Do NOT pin versions that conflict with Expo expectations
